# Quickstart guide

Get AlignTrue running in under 60 seconds with three commands.

## Prerequisites

- **Node.js 22+** - [Download](https://nodejs.org/)
- **An AI coding agent** - Cursor, GitHub Copilot, Claude Code, or any of [28+ supported agents](/docs/04-reference/agent-support)

## Quick start (3 commands)

### 1. Initialize

```bash
cd your-project
aligntrue init
```

This detects your agents and creates:

- `AGENTS.md` - Your rules file (editable markdown)
- `.aligntrue/config.yaml` - Configuration
- `.aligntrue/.rules.yaml` - Internal format (auto-generated, don't edit)

### 2. Sync to your agents

```bash
aligntrue sync
```

This generates agent-specific files:

- `.cursor/rules/aligntrue.mdc` (for Cursor)
- `AGENTS.md` (for Copilot, Claude, Aider, etc.)
- `.vscode/mcp.json` (for VS Code MCP agents)

### 3. Customize your rules

Open `AGENTS.md` and edit the starter rules for your project:

```markdown
---
id: "project-rules"
version: "1.0.0"
summary: "Project rules and standards"
---

# Project Rules

## Use TypeScript Strict Mode

Enable strict mode in tsconfig.json for better type safety. This catches more errors at compile time and makes refactoring safer.

Set these in tsconfig.json:

- noImplicitAny: true
- strictNullChecks: true
- strictFunctionTypes: true
```

**That's it!** Run `aligntrue sync` again whenever you update your rules.

## What you get

After running `aligntrue sync`, you'll have agent-specific rule files:

**For Cursor:**

- `.cursor/rules/aligntrue.mdc` - Rules in Cursor's native format

**For GitHub Copilot, Claude Code, Aider, and others:**

- `AGENTS.md` - Universal markdown format

**For VS Code with MCP:**

- `.vscode/mcp.json` - Model Context Protocol configuration (if enabled)

## Verify it works

Let's confirm your agent is using the rules:

### Cursor verification

1. Open Cursor Settings (Cmd/Ctrl + ,)
2. Navigate to Features > Cursor Rules
3. Enable "Rules folder" option (if not already enabled)
4. Restart Cursor
5. Open a file and start a chat
6. Ask: "What are the TypeScript rules for this project?"
7. Cursor should reference your rules from `AGENTS.md`

**Expected:** Cursor responds with your project's rules

**Problem?** See the [troubleshooting guide](/docs/05-troubleshooting).

### GitHub Copilot / Claude Code verification

These agents automatically read `AGENTS.md` from your project root - no setup needed!

1. Open a file
2. Start typing code that might violate a rule
3. Watch for suggestions that follow your rules

**Problem?** Make sure `AGENTS.md` exists in your project root:

```bash
ls -la AGENTS.md  # Should show the file
```

### Common issues

**Rules not showing in my agent:**

1. Verify files were created: `ls .cursor/rules/` or `ls AGENTS.md`
2. Restart your IDE completely
3. Check agent settings to ensure rules are enabled
4. Run `aligntrue sync` again to regenerate files

**Still not working?** See the [troubleshooting guide](/docs/05-troubleshooting).

## How it works

AlignTrue uses a simple three-step process:

1. **Write rules** - Edit `AGENTS.md` in natural markdown
2. **Generate IR** - AlignTrue converts to internal format (`.aligntrue/.rules.yaml`)
3. **Export** - Rules sync to all configured agents (`.cursor/*.mdc`, `AGENTS.md`, etc.)

**Workflow:**

```
AGENTS.md (you edit) → .aligntrue/.rules.yaml (auto-generated) → Agent exports
```

**Key concepts:**

- **Rules** - Guidelines for AI agents (e.g., "use TypeScript strict mode")
- **Sync** - Convert rules into each agent's native format
- **Agents** - AI coding assistants (Cursor, Copilot, Claude Code, etc.)

See the [Glossary](/docs/03-concepts/glossary) for complete terminology.
